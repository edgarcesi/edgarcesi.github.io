Subject

The aim of this project was to develop a communication simulator.

In order to carry out this project several steps were necessary, such as the specification of the need and the software architecture.

The software was to test and validate the communication protocols of a field information feedback system for the RATP. It was to simulate field actions and transmit the various corresponding messages on the serial port.

Operation

Connection to the serial port

When the application is launched, it will first be necessary to connect to the serial port where it will be necessary to insert an available port number, a speed and the mode (emission / transmission)

Composition of a message

When a message is sent it follows a very specific architecture which is composed of 5 bytes per message sent.

Sending messages

After connecting to the different serial ports, two interfaces will be displayed. The first one, which is the receiving part (in a thread), will wait and as soon as it receives information it will display it on the interface.
The information is sent to radio buttons.
The second interface which will be displayed is the one which allows the transmission of messages also in another thread.
